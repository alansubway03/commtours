# 一步一步：其他旅行社抓取教學

用瀏覽器開發者工具找出「行程卡」的 HTML 結構，再改爬蟲的 selector，就能讓 EGL、金怡、縱橫遊等站抓到真正的旅行團資料。

---

## 第一步：用 Chrome 打開旅行社的「行程列表頁」

1. 打開 **Google Chrome**。
2. 在網址列輸入下面其中一個網址（每次只處理一家）：
   - **EGL 東瀛遊**：https://www.egltours.com/travel/pages/EGL_travel_tour_LongHaultravel/index.html
   - **金怡假期**：https://www.goldjoy.com/tour
   - **縱橫遊**：https://www.wwpkg.com.hk/tour/landing?keywords=長線精選
   - **Jetour 捷旅**：https://www.jetour.com.hk/tour
3. 等網頁**完全載入**（有看到一格格行程卡或列表再繼續）。

---

## 第二步：打開開發者工具並選「一個行程卡」

1. 在網頁**空白處**按右鍵，選 **「檢查」**（或按 `F12`），右側或下方會出現開發者工具。
2. 點開發者工具左上角的 **「選取元素」** 圖示（箭頭＋方框）。
3. 用滑鼠在網頁上**點一下「其中一張行程卡」**（有團名、價錢、報名按鈕的那一塊）。
4. 開發者工具會自動跳到對應的 HTML，通常會反白一段 `<div>...</div>` 或 `<article>...</article>`。

---

## 第三步：找出「整張卡」的 class 或標籤

1. 在反白的那段 HTML 上，**由下往上**看父層（外層的 `<div>`、`<section>`、`<article>`）。
2. 找到**包住「一整張行程卡」**的那一層（同一層會重複出現很多次，每張卡一個）。
3. 看該層的 **class** 或 **標籤名**，例如：
   - `class="tour-card"`
   - `class="product-item"`
   - `class="tour-item"`
   - `<article>`
4. 把這個 **「整張卡的選擇器」** 記下來，例如：`.tour-card` 或 `article`。  
   - 若 class 很長，可以只取前面一段，例如 `[class*="tour-card"]` 表示 class 裡有 `tour-card` 的字串。

**範例**：若包住整張卡的是 `<div class="tour-card">`，選擇器就寫 **`.tour-card`**。

---

## 第四步：在同一張卡裡找出「團名」「連結」「價錢」

在**同一段**「整張卡」的 HTML 裡（不要點到別張卡或頁首頁尾），找：

1. **團名／標題**  
   - 多數在 `<h2>`、`<h3>`、`<h4>` 或帶 `title`、`name` 的 class。  
   - 記下：用什麼標籤或 class，例如 `h3`、`.tour-title`。

2. **「報名」或「查看詳情」的連結**  
   - 點開 `<a href="...">`，確認 `href` 是**單一團的詳情頁**，不是分類頁或首頁。  
   - 記下：例如 `a`、`a.btn-book`、`a[href*="/tour/"]`。

3. **價錢**（若有顯示）  
   - 多數在 class 含 `price`、`amount` 的區塊。  
   - 記下：例如 `.price`、`[class*="price"]`。

以上都是**在這張卡內部**的選擇器，不要選到整頁共用的 header/footer。

---

## 第五步：把結果填進爬蟲設定

1. 用編輯器打開專案裡的 **`scripts/scrape-tours-playwright.ts`**。
2. 找到 **`SITE_SELECTORS`**（約第 31 行），裡面有 `wingon`、`egl`、`goldjoy`、`jetour`、`wwpkg`。
3. 找到你要改的那一家（例如 `egl`），會看到類似：

   ```ts
   egl: [
     { container: ".tour-item", titleSel: "h2, h3, .title, a", linkSel: "a", priceSel: ".price, .amount" },
     ...
   ],
   ```

4. 把**第一組**改成你在第三步、第四步記下的選擇器：
   - **container**：整張卡的選擇器（例如 `.tour-card`）。
   - **titleSel**：團名（例如 `h3` 或 `.tour-title`）。
   - **linkSel**：詳情/報名連結（例如 `a` 或 `a[href*="/itinerary/"]`）。
   - **priceSel**：價錢（例如 `.price`），沒有就保留原樣或刪掉。

5. **存檔**。

---

## 第六步：排除「不是行程」的連結（可選）

若抓到的結果裡還有「人才招聘」「會員登入」等，代表有些連結不該算行程：

1. 在 **`LINK_EXCLUDE`**（約第 64 行）找到該站。
2. 在字串裡加上**不要的網址特徵**，用 `|` 分隔，例如：
   - `egl: "...|/site/career|/trip-packages"`

若有些項目是**用標題**過濾（例如「會員登入」），在 **`EXCLUDE_TITLE_KEYWORDS`**（約第 109 行）的正則裡加上對應關鍵字。

---

## 第七步：執行爬蟲並檢查結果

1. 在終端機（PowerShell 或 CMD）打：

   ```bash
   cd c:\Users\User\tour-compare
   npx tsx scripts/scrape-tours-playwright.ts
   ```

2. 跑完後打開對應的 JSON 看有沒有該站的資料，例如：
   - EGL：`scripts/output/tours-egl.json`
   - 金怡：`scripts/output/tours-goldjoy.json`
   - 縱橫遊：`scripts/output/tours-wwpkg.json`
   - Jetour：`scripts/output/tours-jetour.json`

3. 檢查：
   - **筆數**是否合理（不會只有 1、2 筆或上百筆雜項）。
   - **標題**是不是團名，而不是「立即報名」「會員登入」。
   - **連結**點開是否真的是該團的詳情頁。

若結果不對，回到**第三步、第四步**再對一次 DOM，調整 container / titleSel / linkSel / priceSel，然後重跑第七步。

---

## 第八步：寫入 Supabase（有需要時）

確認 JSON 內容正確後，再寫入資料庫：

```bash
npx tsx scripts/push-tours-to-supabase.ts --upsert scripts/output/tours-egl.json
npx tsx scripts/push-tours-to-supabase.ts --upsert scripts/output/tours-goldjoy.json
npx tsx scripts/push-tours-to-supabase.ts --upsert scripts/output/tours-jetour.json
npx tsx scripts/push-tours-to-supabase.ts --upsert scripts/output/tours-wwpkg.json
```

（永安若也有更新，可同樣加一行 wingon 的 push。）

## 快速對照：要記下的三件事

| 步驟 | 你要在網頁上找到 | 填到 scrape-tours-playwright.ts 的欄位 |
|------|------------------|----------------------------------------|
| 第三步 | 包住「一整張行程卡」的那一層的 class 或標籤 | `SITE_SELECTORS.該站[0].container` |
| 第四步 | 卡內的「團名」在哪個標籤/class | `SITE_SELECTORS.該站[0].titleSel` |
| 第四步 | 卡內的「詳情/報名」連結 | `SITE_SELECTORS.該站[0].linkSel` |
| 第四步 | 卡內的「價錢」（可選） | `SITE_SELECTORS.該站[0].priceSel` |

若某一家網頁結構較複雜（例如 SPA、iframe），可把該站的列表頁網址和「你看到的 container / 團名 / 連結」貼給我，我可以幫你寫好對應的 selector 和排除規則。
