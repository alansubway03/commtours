# 自動化從各大旅行社抓取旅行團資料 — 最佳做法

目標：**每天自動**從五家（或更多）旅行社取得旅行團列表，寫入 Supabase，網站自動顯示最新資料。

---

## 一、推薦主線：Playwright + 排程（免 API 費、可完全自動）

### 流程

```
Playwright 爬蟲（每家專用 selector）→ JSON 檔 → push-tours-to-supabase.ts --upsert → Supabase
```

### 優點

- **無額外 API 費用**（不用 Gemini / Claude）
- **可完全自動化**（Windows 工作排程每天跑一次）
- **穩定**：同一套 selector 每天跑結果一致
- **專案已內建**：`scrape-tours-playwright.ts` + `daily-update-tours.ps1`

### 一次手動執行（測試用）

```powershell
cd c:\Users\User\tour-compare
.\scripts\daily-update-tours.ps1
```

會依序：抓五家 → 產出 `scripts/output/tours-*.json` → 對 Supabase 做 upsert。

### 每天自動執行（建議）

1. 確認專案根目錄有 `.env.local`，內含：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
2. 開啟 **Windows 工作排程器** → 建立基本工作：
   - 名稱：例如「每日更新旅行團」
   - 觸發：每日，時間自訂（例如 凌晨 2:00）
   - 動作：啟動程式
     - 程式：`powershell.exe`
     - 引數：`-ExecutionPolicy Bypass -NoProfile -File "c:\Users\User\tour-compare\scripts\daily-update-tours.ps1"`
     - 開始於：`c:\Users\User\tour-compare`

之後每天到點就會自動抓五家並更新 Supabase。

---

## 二、提升資料品質（每家都要對一次）

目前 **永安（日本列表頁）** 較穩；其餘四家若出現錯鏈或錯欄位，需要「對 DOM 調 selector」。

### 做法

1. 用瀏覽器打開該旅行社的**實際旅行團列表頁**（不是首頁、不是分類選單）。
2. F12 → 檢查「一個旅行團卡片」的 HTML：外層容器 class、標題、連結、價錢的 class 或路徑。
3. 在 `scripts/scrape-tours-playwright.ts` 的 `SITE_SELECTORS` 裡，為該家補上或修改：
   - `container`：一張卡片的容器
   - `titleSel` / `linkSel` / `priceSel`：標題、詳情連結、價格
4. 若該站列表在「子頁」（例如選國家後才出列表），在 `SITES` 裡把該家的 `url` 改成**直接是列表頁**的網址。
5. 在 `LINK_EXCLUDE` 裡排除該站的分類、導覽連結，只保留真正的行程連結。

調好後再跑一次 `.\scripts\daily-update-tours.ps1` 檢查結果。

## 三、選配：OpenClaw + 付費 Gemini（難搞的站用 AI 補強）

適合：某幾家 DOM 很亂、改版頻繁，用 selector 很難維護時。

- **設定**：見 `scripts/OPENCLAW_GEMINI_QUICKSTART.md`，用 **Gemini 付費方案** 避免 rate limit。
- **用法**：對「難抓的那一家」用 OpenClaw 下自然語言指令，要它用 browser 開該站、抓 N 筆旅行團、輸出 JSON。
- **匯入**：把回傳的 JSON 整理成與 `tour` 表對應的欄位（補 `agency`、`type`、`region`、`affiliate_links` 等），再用 `push-tours-to-supabase.ts --upsert` 寫入。

建議當作**輔助**：主線仍用 Playwright 每日自動跑；只有少數站用 OpenClaw 手動或每週補一次。

---

## 四、新增第六家、第七家旅行社

1. 在 `scripts/scrape-tours-playwright.ts` 的 `SITES` 加一筆（`name`, `agency`, `url`, `affiliateKey`）。
2. 在 `SITE_SELECTORS` 為該家加對應的 `container` / `titleSel` / `linkSel` / `priceSel`（必要時加 `LINK_EXCLUDE`）。
3. 在 `scripts/daily-update-tours.ps1` 的 `$files` 陣列加一個檔名，例如 `tours-新家.json`。
4. 跑一次 `daily-update-tours.ps1` 測試，再交給排程即可。

---

## 五、總結表

| 項目           | 建議做法 |
|----------------|----------|
| **主線自動化** | Playwright + `daily-update-tours.ps1` + Windows 每日排程 |
| **資料品質**   | 每家對 DOM 調好 selector 與列表頁 URL，必要時加 LINK_EXCLUDE |
| **擴充家數**   | 在 SITES / SITE_SELECTORS / daily 的 $files 加一筆 |
| **難抓的站**   | 選配：OpenClaw + 付費 Gemini，手動或偶爾跑，結果再 push --upsert |

依上述做法，即可在**免額外 API 費**的前提下，用一套腳本與排程自動化從各大旅行社拿取旅行團資料並寫入 Supabase。
