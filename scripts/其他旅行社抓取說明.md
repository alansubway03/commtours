# 其他旅行社抓取旅行團資料 — 做法說明

## 現況

- **永安**：已用多個長線地區 URL + 專用 selector，可穩定抓到產品卡與每日期狀態。
- **EGL、金怡、Jetour、縱橫遊**：目前多數抓到的是導覽/分類連結（如「人才招聘」「自由行套票」「已成團推介」），不是行程產品卡。

## 做法一：改 URL + 收緊排除（本次已做）

1. **EGL 東瀛遊**  
   - 改用長線列表頁：`https://www.egltours.com/travel/pages/EGL_travel_tour_LongHaultravel/index.html`  
   - 排除：`/site/career`、`/trip-packages`、`/travel/pages`（只保留行程詳情連結）、標題「立即致電」「人才招聘」「自由行套票」等。

2. **Jetour 捷旅**  
   - 保留 `https://www.jetour.com.hk/tour`，排除連結：`theme-detail`、`studytour`、`bespoke-journey`，避免把活動/學堂當成團。  
   - 可再加長線目的地列表：`/tour/dest/europe`、`/tour/dest/africa` 等（需確認該頁是否為產品卡列表）。  
   - 標題排除：「已成團推介」「捷旅學堂」「自由行套票」等。

3. **金怡假期**  
   - 保留 `https://www.goldjoy.com/tour`。  
   - 排除：`about-us`、`career`、`tour/theme`、`tour/status`、`tour?searchCriteria=`（分類頁），只保留 `tour?code=XXX` 這類行程詳情。  
   - 標題排除：「企業架構」及純地區名（若該地區是分類而非團名）。

4. **縱橫遊 WWPKG**  
   - 改用長線精選：`https://www.wwpkg.com.hk/tour/landing?keywords=長線精選`  
   - 維持排除：`tour/search$`、`/info/`、`populardata` 等，確保只留行程詳情連結。

## 做法二：每站用開發者工具對 DOM（建議之後做）

- 用 Chrome 開上述列表頁，在 Elements 裡看：
  - 真正「一團一卡」的容器 class（例如 `.tour-card`、`.product-item`）。
  - 標題、連結、價錢、出發日等所在節點。
- 在 `scrape-tours-playwright.ts` 的 `SITE_SELECTORS` 為該站加一組「只對產品卡」的 selector，並在 `LINK_EXCLUDE` 排除分類/導覽。

## 做法三：必要時進詳情頁補資料

- 若列表頁只有團名 + 連結，沒有價錢/出發日：
  - 可像永安一樣，對每筆連結再開詳情頁，用 `SCRAPE_DETAIL_DATES=1` 或專用邏輯抓價錢、出發日期、成團狀態。
  - 需注意請求數與該站是否有防爬。

## 本次已改動

- **EGL**：改用長線列表 `travel/pages/EGL_travel_tour_LongHaultravel/index.html`；排除 career、trip-packages、survey 等。目前可抓到約 8 筆，連結為 `itinerary/tour/團號`（正確），但標題多為「✈️ 立即報名」— 需在該頁找產品名稱的 selector 或改從詳情頁補標題。
- **Jetour**：排除 theme-detail、studytour、bespoke、seminar 連結與「旅遊講座」等標題。已可抓到真實行程（如南美兩國、澳洲塔斯曼尼亞、杜拜等）與 `/tour/團號` 連結。
- **金怡**：排除 about-us、career、searchCriteria、`tour?region=`，只留 `tour?code=XXX`。目前仍多為地區分類（tour?region=），需確認金怡列表頁產品卡的連結格式是否為 `tour?code=`；若是 SPA，可能需等 AJAX 載入後再抓。
- **縱橫遊**：改用 `tour/landing?keywords=長線精選`；排除 `/misc/`、`/member/` 與「私人組團」「學術交流團」「會員登入」。目前仍抓到導覽，需用開發者工具看 landing 頁上行程卡的 container/連結結構，再補專用 selector。
- **共通**：在 `EXCLUDE_TITLE_KEYWORDS` 加入四站常見導覽/活動文字，過濾非行程項目。

## 建議下一步

1. **EGL**：在長線列表頁用開發者工具找「團名」所在元素（例如每張卡上的 h2/h3），在 `SITE_SELECTORS.egl` 加一組優先使用的 selector，讓標題是團名而非「立即報名」。
2. **金怡**：確認 `/tour` 頁產品卡連結是 `tour?code=XXX`；若一載入是地區篩選，可改為直接抓 `tour?region=Europe` 等頁，或等前端渲染完成後再取 `a[href*='code=']`。
3. **縱橫遊**：在 `tour/landing?keywords=長線精選` 用開發者工具找行程卡的外層 class 與內部的標題/連結，為 wwpkg 加專用 selector，並在 LINK_EXCLUDE 排除非行程的連結。
4. **Jetour**：已可用的前提下，可再加多頁 URL（如 `/tour/dest/europe`）以增加筆數，並確認該頁是否為產品列表。
